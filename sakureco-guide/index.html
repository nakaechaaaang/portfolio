サクレコで
画面を録画して、すぐ共有

サクレコ（SAKU-RECO）は、ブラウザだけで画面を録画して、その場で共有リンクを作れるツールです。登録不要・無料・最長2時間。インストールも要りません。

🎥 サクレコを開く

What is it

サクレコでできること

「画面の操作を見せながら説明したい」「長い手順を口頭で伝えたい」——そんなときに、録画してリンクを送るだけで伝わります。

⏺️
登録不要ですぐ録画サイトを開いてボタンを押すだけ。アプリのインストールも不要。
⏱️
最長2時間・無制限長い手順や打ち合わせも途中で切れずに録画できます。
🔗
録画したら即リンク停止すると共有リンクが発行。コピーして送るだけ。
🙂
顔・声も入れられるカメラの映像を左下に丸く重ね、マイク＋画面音声をミックス。
✂️
必要な所だけ共有開始・終了を指定すれば、その範囲だけ見せられます（無劣化）。
📱
スマホでも視聴OKmp4で録画すれば、相手はスマホでも見られます。

Before you start

はじめる前に

  • ブラウザは Chrome または Edge がおすすめです（録画機能が安定します）。
  • 顔や声を入れたい場合は、カメラ・マイクを準備（後から「許可」を押すだけ）。
  • 画面の音（動画やアプリの音）も入れたいときは、画面共有のときに「システム音声を共有」にチェックします。
💡

サクレコはブラウザの中だけで動くので、ソフトのインストールやアカウント登録はいりません。

How to record

録画のしかた（ステップ）

1

サクレコを開く

sakureco.surge.sh を Chrome / Edge で開きます。

2

画質を選ぶ

標準と高画質の2択。長い録画（1〜2時間）は、軽くて容量も少ない標準がおすすめです。短くてキレイに見せたいときだけ高画質に。

3

「録画開始」を押して、映す範囲を選ぶ

ボタンを押すと、ブラウザが「どれを共有しますか？」と聞いてきます。画面全体／ウィンドウ／タブから、録りたいものを選びます。音も入れたいときは「システム音声を共有」にチェック。

4

（任意）カメラ・マイクを許可

顔出し・声入れをするなら、カメラとマイクを「許可」します。自分の映像が左下に丸く重なって表示されます。マイクと画面の音は自動でミックスされ、音ズレしません。

5

話しながら操作する

あとは画面を操作しながら説明するだけ。録画中はそのまま進めてOKです。

6

「停止」で終了 → 自動アップロード

終わったら停止ボタンを押します。録画が自動でアップロードされます（長い動画は少し時間がかかります）。

7

共有リンクをコピーして送る

アップが終わると共有リンクが発行されます。コピーして、メールやチャットで送るだけ。相手は登録不要で見られます。

Share & watch

共有・視聴のしかた

共有リンクを開くと、視聴ページで動画がそのまま再生されます。受け取った人は、アカウントを作らずに見られます。

  • リンクを送るだけ。相手はクリックして再生するだけです。
  • スマホでも視聴できます（mp4で録画した場合）。
  • 動画は読み込みながら再生されるので、長尺でも頭出しがスムーズです。

Trim

必要な部分だけ見せる（トリム）

「この5分だけ見てほしい」というときは、開始時間と終了時間を指定した共有リンクを作れます。元の動画はそのまま残り、編集の処理待ちゼロ・画質の劣化なしで、指定した範囲だけが再生されます。

✂️

共有リンクに「開始秒」「終了秒」を付けるだけ。動画を切り出して作り直す必要はありません。

Quality

画質と長さの目安

画質こんなとき容量の目安
標準（約2.5Mbps）長い手順・打ち合わせ・1〜2時間の録画2時間で約2.3GB
高画質（約8Mbps）短く、文字や細部までくっきり見せたいとき標準より大きめ

迷ったら標準でOK。長尺でも軽く、共有もスムーズです。

Troubleshooting

うまくいかないとき

Q. 録画ボタンが反応しない／画面が選べない

ブラウザを Chrome か Edge にしてください。一部のブラウザは画面録画に対応していません。

Q. iPhone で再生できない

webm 形式の動画は iPhone の Safari で再生できないことがあります。mp4 に対応したブラウザ（Chrome / Edge）で録画すると、スマホでも見られます。

Q. 動画が途中で止まる・再生されない

「Claude in Chrome」などの拡張機能が動いていると、再生が止まることがあります（サクレコ側の不具合ではありません）。拡張機能をオフにするか、別のブラウザで開いてみてください。

Q. 音が入っていない

画面を選ぶときに「システム音声を共有」にチェックが入っているか、マイクが「許可」になっているかを確認してください。

⚠️

録画した動画は容量に上限があり、古いものは整理される場合があります。大事な録画は手元にも保存しておくと安心です。

Setup（上級）

自分用にサクレコを構築する（設定方法）

サクレコは フロント（録画・視聴ページ）＋ Cloudflare Worker ＋ R2ストレージ でできています。自分専用に立てたい人向けに、設定の流れをまとめます。

🧩

仕組み：ブラウザで録画 → 64MB分割で Worker にアップロード → R2 に保存。視聴ページは Worker から Range対応（途中再生OK）で動画を配信します。2時間級の長尺を“無料”で成立させる構成です。

手順（ゼロから）

1

Cloudflareにサインアップ（無料）

dash.cloudflare.com/sign-up でメールアドレスとパスワードを入力 → 届いた確認メールのリンクで認証します。ドメイン購入などは不要です。

2

R2を有効化してバケットを作る

ダッシュボード左の 「R2」 →「Purchase R2 Plan / R2を始める」で有効化 → 「Create bucket」で保存用バケットを作成（名前は任意。例：sakureco）。R2は10GB無料・配信転送量も無料です。

💳

R2の有効化時にクレジットカードの登録を求められます。無料枠（10GB・転送量0円）の範囲なら請求は発生しませんが、カード登録自体は必須です（初見でつまずきやすいポイント）。

3

操作ツール（wrangler）を準備

Node.js（nodejs.org の「LTS」）を入れたら、ターミナルで：

npm install -g wrangler
wrangler login … ブラウザが開くので「Allow」で連携

4

Worker（保存・配信サーバー）をデプロイ

Worker は「録画ファイルを64MB分割で受け取って R2 に保存」「/file/ で Range対応配信」を担当します。R2バケットを Worker にバインドして wrangler deploy で公開。デプロイ後、配信URLは https://<あなたのWorker>.workers.dev/file/rec/… の形になります。

5

フロントに自分のWorker URLを設定

録画ページ（index.html）のアップロード先と、視聴ページ（watch.html）が許可する配信ホストを、自分のWorker URLに合わせます。視聴ページは「自分のWorkerホスト＋ /file/rec/ 以外のURLを拒否」する作りなので、ここを自分のドメインにするのが安全上のキモです。

6

フロントを surge で公開

録画・視聴ページのフォルダで公開します。
surge ./ <あなたの名前>.surge.sh

7

運用：容量の管理

標準画質で2時間 ≈ 2.3GB → 10GB で約4本が目安。いっぱいになったら古いものから削除します。

一覧：wrangler r2 object list <バケット名>
削除：wrangler r2 object delete "<バケット名>/rec/ファイル名"

設定のコツ・注意

  • 録画形式は mp4優先 → webmフォールバック（iPhone視聴の互換性のため）。
  • アップロードは 64MB分割＋パート毎リトライ。Worker無料枠のボディ上限を超えないための工夫です。
  • 画質は標準（約2.5Mbps）／高画質（約8Mbps）の2択。長尺は標準推奨。
⚠️

上記はサクレコの設計メモ（README）をもとにした設定の概要です。Worker のコードやフロントの該当箇所など細部は実際のファイルに合わせて調整してください。実コード（index.html / watch.html / Workerソース）を共有してもらえれば、そのまま使える手順に書き起こせます。
